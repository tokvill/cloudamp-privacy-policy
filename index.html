Technical Data Handling, Storage Permissions, and Cloud Integration

For fully functional audio playback, file downloading, and cloud storage streaming, the Application strictly adheres to the following technical data handling procedures and permissions:

  • Local Storage & Audio Access (READ_MEDIA_AUDIO / READ_EXTERNAL_STORAGE): Storage permissions are requested strictly and exclusively for scanning, cataloging, displaying album art, playing locally stored audio files (such as music, podcasts, and audiobooks), and saving user-downloaded audio tracks to device memory. The Application does not access, scan, or process non-audio personal files, documents, or photos.
  • Local Data Isolation: All user-generated data — including playlists, equalizer presets, playback history, timestamps, and custom organization structures — is stored exclusively within the local database on the user's device. This data is isolated locally and is never transmitted to or stored on external servers owned by the Service Provider.
  • User-Initiated File Downloading: The Application allows users to download audio files from their connected cloud storage accounts directly to their mobile device for offline playback. The Application does not possess the capability to delete, rename, or move existing files on the user's device. Any file download operation occurs strictly by direct, explicit user initiative and is never performed in the background without the user's explicit action.
  • Cloud Storage Integration via OAuth 2.0 (Google Drive, Dropbox, Yandex Disk): To enable audio streaming and downloading directly from personal cloud storage, the Application integrates with Google Drive, Dropbox, and Yandex Disk via industry-standard, secure OAuth 2.0 authorization protocols (or official provider SDKs). The Application receives only scoped access tokens required to list, stream, and download audio files from user-designated folders. The Application never receives, views, or stores users' primary login credentials (usernames or passwords).
  • Cloud Storage Integration via WebDAV (Mail.ru Cloud): Connection to Mail.ru Cloud is implemented using the standard WebDAV network protocol. Any authentication credentials or app-specific passwords entered by the user are stored strictly within the device's protected local database. These credentials are utilized exclusively for direct, encrypted communication between the user's device and Mail.ru servers, and are never transmitted to, viewed by, or stored on external servers owned by the Service Provider.
  • Direct P2P / Peer-to-Cloud Streaming Architecture: Audio streaming and file downloading occur directly between the user's device and the servers of the selected cloud provider (Google Drive, Dropbox, Mail.ru, or Yandex Disk) over encrypted HTTPS connections. Audio streams and personal data are never routed through, mirrored, or cached on any intermediary or proxy servers of the Service Provider.
  • Local Stream Caching: To ensure smooth playback and enable seamless listening, the Application may temporarily store audio stream fragments (cache) within dedicated, protected local device memory. Cached audio data is utilized exclusively by the Application's internal audio engine and is never used for user tracking or behavioral analytics.
  • Crash Log Reporting via paste.rs: To identify, diagnose, and fix technical issues or application crashes, the Application utilizes the external service paste.rs. In the event of an application crash, anonymized technical error logs (including stack traces, device model, and operating system version) may be transmitted to paste.rs. These crash reports contain strictly technical debugging information and do not include personally identifiable information, stored audio files, login credentials, or user library data.
  • No Acoustic or Biometric Data Collection: The utilization of the Application's audio engine and system audio interfaces (including equalizer processing and background playback) is dedicated solely to audio signal processing and output. The Application strictly does not record audio from the device's microphone, nor does it collect, analyze, or process acoustic or biometric data.
